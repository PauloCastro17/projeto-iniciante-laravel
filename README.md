## ⏳ "TimeLine" do projeto
- **23/03** — Dia que criei o projeto que consiste em um Habit Tracker (rastreador de hábitos). Inicei 
entendendo como criar meu primeiro projeto e principalmente entendo o conceito da estrutura do Laravel. 
Entendi como criar controladores por comando e como eles são utilizados de maneira a se comunicar com o 
gerenciamento de rotas. Criei uma view básica para ver como se comporta a renderização e tammbém criei uma 
pasta de components para poder criar "layouts" repetitivos(o tal do components). E dei inicio a entender como 
funciona as migrations do Laravel, uma ferramenta útil e principalmente versátil para criação de tabelas.
<br><br>
- **24/03** — Comecei dando os primeiros passos entendendo como funciona o login e o logout com laravel, entendo ainda mais 
o gerenciamento de rotas e vendo como se utilizar os "middlewares". Continuei na criação de telas e criei o inicio das telas 
de dashboard, login e a funcionalidade de logout. O login já está validando os dados no banco de dados. Também aprendi uma 
funcionalidade que permite inserir "dados de teste" diretamente no banco de dados que são os **Seedeers**, funcionalidade no 
qual faz parte do contexto das migrations.
<br><br>
- **25/03** - Forma de login reformulada, anteriormente as credenciais eram validadas dentro do controller, mas é melhor que 
seja criado uma classe propria, tal classe que estende da **FormRequest**. Mesma coisa foi feita para a parte de cadastro do 
usuário. Começo da criação de tabelas de hábitos e o log de hábitos e também foi feito pequenas alterações de front.
